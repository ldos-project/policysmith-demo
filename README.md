# PolicySmith Leaderboard

A live leaderboard for the Vulcan cache eviction heuristic competition.

## Quick start

```bash
# From /home/ardula/leaderboard
bash start.sh
```

This does all four steps below in order. Open http://localhost:5000 when it finishes.

---

## Step by step

### 1. Build the simulator

```bash
cd policysmith-demo
mkdir -p build
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j --target run_algo.o
cd ..
```

### 2. Run baseline simulations

Runs 9 classical algorithms (FIFO, LRU, LFU, ARC, S3-FIFO, …) across all
10 traces × 3 cache sizes (270 total simulations). Takes a few minutes.

```bash
bash baselines.sh
```

Produces two files:
- `fifo_baselines.json` — FIFO hit rates used as the MRR denominator
- `baselines_all.json` — all classical algorithm results

### 3. Install dependencies

```bash
pip install flask
```

### 4. Seed the database

`seed_baselines.py` pre-populates the DB with two sets of entries:

**Classical baselines** (from `baselines_all.json`, generated in step 2):
- FIFO, LRU, LFU, LFUDA, ARC, S3-FIFO, CLOCK, WTinyLFU, LHD
- Each gets full 30-scenario results and an MRR vs FIFO score
- These appear in the leaderboard under submitter "Baseline / System"

**Vulcan heuristics** (from `policysmith-demo/results.json`):
- The LLM-evolved policies from the demo run (LRU seed + iterations 0–4)
- Note: `results.json` only stores the w106@10pct training score; the other
  29 scenarios are filled with 0.0. To get full coverage, run `full_eval.py`
  on each `.cpp` snippet and re-submit.

```bash
python3 leaderboard-webapp/seed_baselines.py
```

To re-seed from scratch (e.g. after re-running baselines):

```bash
rm leaderboard-webapp/leaderboard.db
python3 leaderboard-webapp/seed_baselines.py
```

To add more Vulcan heuristics manually, use `full_eval.py` which posts
directly to the running server — no need to touch the DB directly.

### 5. Start the server

```bash
python3 leaderboard-webapp/app.py
# or: PORT=8080 python3 leaderboard-webapp/app.py
```

Dashboard: http://localhost:5000

## Submitting a heuristic

Participants use `full_eval.py` from the repo root:

```bash
python3 full_eval.py path/to/my_heuristic.cpp --server http://localhost:5000
```

It will:
1. Compile the heuristic and run all 30 simulations
2. Print a results table with a local MRR preview
3. Ask for name, group, heuristic name, and a one-line description
4. POST everything to `/api/submit`

MRR (mean ratio vs FIFO) is computed server-side on submission.

To evaluate without submitting:

```bash
python3 full_eval.py path/to/my_heuristic.cpp --no-submit
```

## API

| Method | Path | Description |
|--------|------|-------------|
| `GET`  | `/` | Dashboard |
| `POST` | `/api/submit` | Submit evaluation results |
| `GET`  | `/api/leaderboard` | JSON leaderboard (filterable) |
| `GET`  | `/api/submission/<id>` | Full detail for one entry |
| `GET`  | `/api/fifo_baselines` | FIFO baseline values |

`/api/leaderboard` query params: `sort` (mrr / mean_byte_hr / mean_obj_hr),
`trace` (e.g. w106), `size` (1pct / 3pct / 10pct), `limit`.

## File layout

```
leaderboard/
  full_eval.py          # evaluation + submission script (give this to participants)
  baselines.sh          # generates fifo_baselines.json and baselines_all.json
  start.sh              # one-shot setup + launch
  fifo_baselines.json   # generated by baselines.sh
  baselines_all.json    # generated by baselines.sh
  leaderboard-webapp/
    app.py              # Flask server
    seed_baselines.py   # seeds DB from baselines_all.json + results.json
    leaderboard.db      # SQLite DB (created on first run)
    requirements.txt
    templates/
      index.html        # dashboard UI
  policysmith-demo/     # git submodule with simulator + traces
```
